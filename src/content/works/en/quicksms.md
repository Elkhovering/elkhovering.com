## Context

QuickSMS is a marketplace for virtual phone numbers used in one-time SMS verification — the kind you need when a service blocks VoIP numbers or you need a temporary identity. Core audience: developers, digital marketers, privacy-conscious users.

## Problem

The existing interface was clunky and slow, with no clear inventory state and a confusing checkout. Users were dropping off before completing purchase because they couldn't tell if a number was actually available.

## My role

Product design and frontend implementation. I redesigned the purchase flow, built the real-time availability UI, and shipped the frontend solo.

## Process

Reduced the purchase to three steps: pick country → pick service → pay. Built live inventory indicators so users could see number availability without refreshing. Designed clear error states for when a number gets taken mid-checkout. Handled the edge cases: refunds, number expiry, failed OTP delivery.

## Outcome

Checkout completion rate improved measurably post-launch. Support tickets about "where's my number" dropped significantly. The platform's repeat-buyer rate increased — the cleaner UX built trust.
