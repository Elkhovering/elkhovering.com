## Context

XMR Crash is an online casino running the Crash format — a multiplier climbs until it crashes, players cash out before it falls. Built on Monero for provably fair gameplay and full payment privacy. The audience values anonymity above everything else.

## Problem

Crash games on other platforms feel either too garish (casino-kitsch) or too bare (developer side-project). XMR Crash needed a dark, restrained interface that built tension through design, not noise.

## My role

Art direction, UI design, and frontend. I designed the visual language, built the real-time game UI, and handled the WebSocket-driven state updates.

## Process

Built the tension through typography and timing: the multiplier counter is the hero element, everything else is secondary. Dark palette with a single accent for the crash event. The cash-out button is large and unambiguous — no friction when adrenaline is running. Bet history shows session context without crowding the main view.

## Outcome

The restraint paid off: the site became a reference in Monero-casino communities for how the interface should feel. Session lengths were long; the UX friction was low enough that users focused on the game, not the interface.
